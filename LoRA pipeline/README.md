# Qwen Math SFT Pipeline

## Structure

```
qwen_sft/
├── config.py
├── requirements.txt
├── data/
│   ├── training_data.json     ← your input file
│   └── processed/             ← auto-generated by step 1
│       ├── train.jsonl        (70%)
│       ├── val.jsonl          (15%) ← used during training to pick best checkpoint
│       └── test.jsonl         (15%) ← only touched in final evaluation
├── scripts/
│   ├── 01_prepare_data.py
│   ├── 02_train.py
│   └── 03_evaluate.py
└── outputs/
    ├── checkpoints/
    ├── best_checkpoint.txt
    └── results/
        ├── eval_val.json
        └── eval_test.json
```

## Run order

```bash
pip install -r requirements.txt

# 1. Set MODEL_NAME in config.py

# 2. Format data and create splits
python scripts/01_prepare_data.py

# 3. Train (monitors val loss, saves best checkpoint)
python scripts/02_train.py

# 4a. Sanity check on val split
python scripts/03_evaluate.py --split val

# 4b. Final evaluation on test split (do this once, at the end)
python scripts/03_evaluate.py --split test
```

## Split roles

| Split | Size | Purpose |
|-------|------|---------|
| train | 70%  | Model sees this during training |
| val   | 15%  | Picks best checkpoint — checked after each epoch |
| test  | 15%  | Final accuracy number — run once at the very end |

**Never train on val or test. Never look at test results until training is fully done.**

## Answer types handled

| Type | Example answer | How it's scored |
|------|----------------|-----------------|
| Multiple choice | `"C"` or `"H"` | Letter match |
| Free-response | `["64.71", "11.74"]` | All parts must match with numerical normalization (8.0 == 8)|
